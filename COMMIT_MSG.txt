SPRINT 4 · Modelo v2.0 (refactor mayor)

Tras validar la idea con perfiles del sector, se rediseno el modelo:
- Las certificaciones las crea el constructor por demanda (no se predefinen hitos)
- El promotor deposita un porcentaje asegurado (15-40, default 30)
- Factura obligatoria por certificacion
- Anexos formales firmados por las partes para modificaciones

Incluye:
- Migraciones SQL: schema v2 + 10 RPCs (sf_create_pact_v2,
  sf_pact_fund_initial, sf_constructor_create_cert, sf_addendum_create,
  sf_addendum_sign, ...)
- Wizard de creacion rediseñado a 4 pasos con slider del deposito
- Detalle del pacto: widget de deposito (estado/alerta/CTAs), timeline
  de certificaciones con badges de factura/doc/version, seccion de anexos
- 5 bottom-sheets para acciones del modelo v2 (depositar inicial,
  reponer, crear cert, proponer anexo, firmar anexo)
- Dashboards conectados a datos reales con loading skeletons
  (sf_get_dashboard_data)
- 3 hotfixes de schema mismatch (activated_at -> finalized_at,
  milestone_id -> related_milestone_id, validated_by_tech ->
  approved_by_tech)
